Статистика протокола STP/RSTP

Статус моста:
  • Состояние STP/RSTP - активность протокола на коммутаторе.
  • Статус корневого моста - коммутатор является корневым или нет.
  • Протокол - используемый протокол (STP или RSTP).
  • MAC корневого моста - MAC адрес корневого коммутатора.
  • Приоритет корневого моста - приоритет корневого коммутатора.
  • Корневой порт - порт направленный в сторону корневого коммутатора.
  • Путь до корневого коммутатора - стоимость пути до корневого коммутатора.
  • MAC назначенного коммутатора - адрес назначенного коммутатора.
  • Приоритет назначенного коммутатора - приоритет назначенного коммутатора.
  • Bridge max age - время ожидания моста в секундах, по истечению которого он сам высылает сообщение о перестроении сети.
  • Bridge Hello Time - период рассылки BPDU пакетов в секундах.
  • Forward Delay Time - задержка перехода состояний портов из состояния прослушивания (Listening) и обучения (Learning) в состояние пердачи (Forwarding) (в секундах).
  • Время с последнего перестроения сети - время прошедшее с последнего изменения топологии в секундах.
  • Число перестроений сети - общее количество изменений в топологии.

    Статус портов
  • Порт - порт коммутатора.
  • Состояние - активность протокола STP/RSTP на порту.
  • Скорость/Дуплекс
  • Линк - состояние линка на порту.
  • Состояние порта - состояние порта. Порт может находиться в следующих состояниях:
    STPRSTP
    ForwardingForwarding
    DisabledDiscarding
    ListeringDiscarding
    LearningLearning
    BlockingDiscarding
  • Роль порта - роль порта. Возможны следующие роли порта:
    1. Disabled
    2. Root
    3. Designated
    4. Alternate
    5. Backup
  • Приоритет порта - приоритет порта.
  • Цена пути - стоимость пути через порт.
  • P2P - соединение точка-точка.
  • Edge - граничный порт.
  • Forward transitions - число переходов в состояние Forwarding.
